# Simple_ETL

ETL-проект на базе **Apache Airflow**, реализующий инкрементальную загрузку данных из операционной базы данных PostgreSQL в промежуточный слой (MRR) с использованием механизма **High Water Mark (HWM)**. Проект также ведет журнал выполнения ETL-процессов в отдельном хранилище (DWH).

## Возможности

- Инкрементальная загрузка данных (High Water Mark)
- Upsert данных через `ON CONFLICT DO UPDATE`
- Автоматическое обновление HWM после успешной загрузки
- Логирование успешных и неуспешных запусков ETL
- Оркестрация процессов с помощью Apache Airflow
- Использование PostgreSQL в качестве источника, приемника и DWH

# Основные компоненты

## 1. Инкрементальная загрузка

Для каждой таблицы хранится последнее успешно обработанное значение поля `updated_at`.

Перед загрузкой выполняется запрос вида:

```sql
SELECT *
FROM source_table
WHERE updated_at > last_loaded_at;
```

После успешной обработки High Water Mark обновляется.

---

## 2. Upsert

Запись данных выполняется через конструкцию

```sql
INSERT ...
ON CONFLICT (...)
DO UPDATE
```

что позволяет:

- добавлять новые записи;
- обновлять изменившиеся;
- избежать дублирования данных.

---

## 3. High Water Mark

Используется таблица

```text
dwh_high_water_mark
```

в которой хранится:

- название таблицы;
- время последней успешной загрузки.

---

## 4. Логирование

После выполнения каждой задачи в таблицу `etl_logs` записываются:

- DAG;
- задача;
- статус;
- время начала;
- время окончания;
- длительность;
- текст ошибки (при наличии).

Используются callbacks Airflow:

- `on_success_callback`
- `on_failure_callback`

В Airflow должны быть настроены следующие Connections:

| Connection ID | Назначение |
|---------------|------------|
| postgres_operational | Источник данных |
| postgres_mrr | Слой MRR |
| postgres_dwh | DWH (HWM + ETL Logs) |

# Принцип работы

Для каждой таблицы выполняется следующий алгоритм:

1. Получение последнего High Water Mark.
2. Чтение новых данных из операционной БД.
3. Выполнение Upsert в MRR.
4. Обновление High Water Mark.
5. Логирование результата.

---

Получить HWM

        │
        ▼
 Upsert в MRR
        │
        ▼
Обновить HWM
        │

Записать лог выполнения

# Масштабирование

Для добавления новой таблицы необходимо:

1. Создать функцию загрузки с вызовом `load_table_to_mrr()`.
2. Указать:
   - исходную таблицу;
   - целевую таблицу;
   - список колонок;
   - первичный ключ.
3. Добавить новую задачу (`PythonOperator`) в DAG.
